3. warpfs meta — Extended Attributes

Reads user.vfs.* Linux extended attributes on any file. Metadata lives alongside file bytes — content is never modified.

# Set an xattr manually (setxattr not yet in CLI)
$ python3 -c "import os; os.setxattr('src/auth/login.go', 'user.vfs.feature', b'auth-module')"

# Read xattrs via WarpFS
$ warpfs-cli meta src/auth/login.go
feature: auth-module

4. warpfs graph discover — tree-sitter + DuckDB

Parses Go source files with tree-sitter, extracts imports, writes edges.jsonl, initializes DuckDB graph database.

Test fixture

// src/auth/login.go
package auth
import (
    "fmt"
    "os"
    "github.com/org/shared-lib/pkg/utils"
)
func Login() { fmt.Println(os.Getenv("USER")) }

// src/auth/token.go
package auth
func ValidateToken(t string) bool { return len(t) > 0 }

Run

$ warpfs-cli graph discover
Discovered 3 edges across 1 files

$ cat .vfs/graph/edges.jsonl
{"from":"src/auth/login.go","to":"std:fmt","rel":"imports"}
{"from":"src/auth/login.go","to":"std:os","rel":"imports"}
{"from":"src/auth/login.go","to":"pkg:github.com/org/shared-lib/pkg/utils","rel":"imports"}

Edge prefixes: std: for standard library, pkg: for external packages. The graph engine distinguishes them for impact analysis.

DuckDB initialization:

$ ls -la .vfs/graph/
edges.jsonl    211 bytes   ← source of truth
graph.db    798,720 bytes   ← DuckDB persisted indexes and query plans

6. warpfs serve --mcp — MCP Server

Stdio MCP server exposing 3 tools. Any MCP-compatible agent can call them. The server reads xattrs and queries DuckDB in real-time.

Initialize

→ {"jsonrpc":"2.0","id":1,"method":"initialize","params":{...}}
← {"serverInfo":{"name":"warpfs-mcp","version":"0.1.0"},"protocolVersion":"2024-11-05","capabilities":{"tools":{}}}

List tools

→ {"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}
← {"tools":[
{"name":"vfs_get_metadata","description":"Read WarpFS extended attributes for a file.",
"inputSchema":{"properties":{"path":{"type":"string"}},"required":["path"]}},
{"name":"vfs_graph_related","description":"Find files related to the given file via the dependency graph.",
"inputSchema":{"properties":{"path":{"type":"string"}},"required":["path"]}},
{"name":"vfs_graph_stats","description":"Get summary statistics about the dependency graph.",
"inputSchema":{"properties":{},"type":"object"}}
]}

Call vfs_get_metadata

→ {"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"vfs_get_metadata","arguments":{"path":"src/auth/login.go"}}}
← {"content":[{"text":"{\"user.vfs.feature\":\"auth-module\"}","type":"text"}]}

The server is fully JSON-RPC compliant — clean exit on stdin close, no zombie processes.

7. What the Agent Sees

An AI coding agent using WarpFS through MCP can:

# 1. Ask: "what metadata does login.go have?"
vfs_get_metadata("src/auth/login.go")
→ {"user.vfs.feature": "auth-module"}

# 2. Ask: "what files is login.go connected to?"
vfs_graph_related("src/auth/login.go")
→ [{path:"std:fmt", relation:"imports"}, {path:"std:os", relation:"imports"}, ...]

# 3. Ask: "how connected is this codebase?"
vfs_graph_stats()
→ {total_edges:3, unique_sources:1, top_dependencies:[...]}

The agent doesn't need grep or manual import tracing. The VFS has already parsed the AST and built the graph. Metadata queries return in microseconds.

9. How to Run It Yourself

# Clone and build
git clone https://github.com/totalwindupflightsystems/warpfs.git
cd warpfs
cargo build

# Create a test workspace
mkdir /tmp/warpfs-test && cd /tmp/warpfs-test
mkdir -p src/mypackage

# Write a Go file with imports
echo 'package mypackage
import ("fmt"; "os")
func Hello() { fmt.Println(os.Getenv("USER")) }' > src/mypackage/hello.go

# Initialize WarpFS
./target/debug/warpfs-cli init

# Set metadata on the file
python3 -c "import os; os.setxattr('src/mypackage/hello.go', 'user.vfs.feature', b'greeter')"

# Read it back
./target/debug/warpfs-cli meta src/mypackage/hello.go

# Discover the graph
./target/debug/warpfs-cli graph discover

# View stats
./target/debug/warpfs-cli graph stats

# Start MCP server
./target/debug/warpfs-cli serve --mcp
# → paste JSON-RPC on stdin, get responses on stdout
