The Flow

Issue → Merge — 12 Steps, Zero Human Bottlenecks

This is what shipping looks like when agents and humans are equals in the platform.

1

Human creates task

Opens an issue in Forgejo, writes a prompt in prompts/feature-name/v1.md, or assigns from the agent task queue.

Human · Forgejo

2

Axiom assembles the swarm

Reads the task, determines required agents (implementer, reviewer, security auditor), checks agent availability and budgets.

Axiom · Agent Swarm

3

Ralph Loop acquires lock + worktree

Prevents conflicts. Creates isolated worktree. Agents load context from Hivemind memory + DuckBrain.

Ralph Loop · Hivemind · DuckBrain

4

Agent writes code

Runs in isolated H4F container. Uses models pinned in AGENTS.yaml. Tool calls logged via LangFuse.

Hermes Agent · H4F Container

5

Agent commits with attestation

Every commit signed with agent key + metadata: prompt-hash, model, context window. Full lineage stored.

Agent · GitReins Protocol

6

GitReins pre-receive hook blocks or passes

Tier 1 (secrets, lint, tests) + Tier 2 (agentic evaluator reads code, runs checks, delivers PASS/FAIL).

GitReins · Forgejo Hook

7

Agent opens PR

PR auto-filled with: prompt, model, confidence score per file, tests run, files touched.

Agent · Forgejo PR

8

Chimera runs multi-model review

Rewriter designs formation: Sonnet reviews logic, DeepSeek audits security, Gemini checks style. Judges merge. Audit verifies.

Chimera · Multi-Model Panel

9

Conscientiousness runs adversarial self-eval

"Are you sure this is correct?" Adversarial agents challenge assumptions, probe edge cases, attempt to falsify claims.

Conscientiousness · Adversarial Agents

10

PromptFoo CI verifies no regression

Runs .promptfoo.yaml against the prompt version used. Blocks merge if quality drops below threshold.

PromptFoo · Forgejo CI

11

Human + Agent co-approve

Branch protection: 1 human review required + 1 agent review required. Both must approve. Agent can veto with evidence.

Human + Agent · Forgejo

12

Merge + Deploy

Code lands on main. Forgejo Pages auto-deploys if it's a site. LangFuse records final trace. Hivemind updates memory.

Forgejo · Pages · LangFuse

What We Could Build

Five Features That Make This An Origin Competitor

The gap between what we have and what Cursor announced yesterday. Each feature maps to existing code.

Build Next · Priority 1

Agent Identity in Forgejo

H4F's known-friends.json → Forgejo OAuth. Agents get real accounts with profiles, SSH keys, avatars. No more masked bot tokens. Axiom's IAM system already models this — extend it to git.

  • ▸ Agent account creation from known-friends registry
  • ▸ Scoped permissions: push to feat/* but not main, open PRs but not merge solo
  • ▸ Agent budgets per repo (max PRs/day, max commits/hour)
  • ▸ Trust escalation: earn merge rights over time based on PR acceptance rate
Build Next · Priority 2

Pre-Flight Cost Estimator

Before Ralph Loop spawns, estimate token burn and ask for budget approval. LangFuse provides the cost data. Hivemind enforces the budget. No more "$1,200 in a 2-hour demo" surprises.

  • ▸ Per-task cost estimate before agent execution
  • ▸ Model routing: cheap models for boilerplate, frontier for reasoning
  • ▸ Per-agent, per-repo, per-sprint cost dashboards
  • ▸ Auto-escalate: 3 failures on cheap model → escalate to frontier
Build Next · Priority 3

Agent-to-Agent PR Negotiation

Two agents disagree in PR comments. Chimera breaks the tie with multi-model analysis. The debate is public, traceable, and the tie-break is auditable. This is the "agent with real authority" feature.

  • ▸ Agent A implements, Agent B reviews — both posting inline comments
  • ▸ Disagreement → Chimera runs multi-model analysis of the disputed code
  • ▸ Agent veto with evidence: "Blocking merge. SQL injection on line 47."
  • ▸ Human spectates or breaks ties. Escalation path when agents deadlock.
Build Next · Priority 4

Prompt Registry + Attestation

Every prompt is source code in prompts/. Every commit links to its prompt version. GitReins hook verifies the link. PromptFoo in CI catches regressions. Full reproducibility.

  • ▸ prompts/ directory in every repo — version-controlled, git-diffable
  • ▸ Commit attestation: "Generated from auth-implement/v3.md · deepseek-v4-pro · 128K ctx"
  • ▸ Prompt → code lineage: "This function from v2. v3 available."
  • ▸ Reproducible generation: re-run exact prompt + model version
Build Next · Priority 5

Agent Marketplace

Discoverable registry of pre-built agents. "I need a Rust security auditor" → search → install → joins repo's agent pool with preconfigured permissions. Per-agent reputation scores from Axiom's adversarial tracking.

  • ▸ Agent profiles: role, model, prompt, permissions, budget
  • ▸ Public metrics: PR acceptance rate, bug rate, review accuracy
  • ▸ One-click install: agent joins repo's agent pool
  • ▸ Bad agents get low trust scores; good agents auto-escalate permissions
How Hermes4Friends Fits

The Agent Hosting + Identity Layer

H4F is the infrastructure that gives every agent its own sandbox. Each agent runs as a Hermes instance in its own Docker container — isolated, with its own API key, model config, tool permissions, and budget.

Identity Registry

known-friends.json becomes the agent identity registry. Each agent: UUID, role, permissions, budget, reputation. Maps directly to Forgejo OAuth for git access.

Isolated Sandbox

Per-agent Docker container with gluetun VPN + dind executor. Agent can't touch other agents' worktrees or secrets. Blast radius contained.

Budget Enforcement

Per-agent API key with spending limits. H4F tracks token usage. Hivemind enforces per-task budgets. Cost overrun = agent paused, human notified.

Scoped Permissions

Agent A: push to feat/*, open PRs, read CI. Agent B (security): read-only, post review comments. Agent C (trusted): merge to main with co-approval. All scoped at the container level.
