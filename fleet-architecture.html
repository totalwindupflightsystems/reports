🏗️ Coding-Hermes Fleet Architecture

Three-tier autonomous orchestration — agreed model between Bane and Hermes, July 2026

TIER 1 Supervisor

"The Architect" — Bane's proxy

  • Decides models, providers, schedules fleet-wide
  • Balances workload across projects
  • Migrates projects between pipelines
  • Fixes broken configs and PAYG leaks
  • Runs as coding-hermes-supervisor cron every 4h
  • Also runs interactively when Bane is present
  • Delivers HTML fleet report to chat after each run

TIER 2 Foremen

"The Foreman" — 1 per project

  • Reads .coding-hermes/tasks.md
  • Picks highest-priority task
  • Compiles prompt through foundry
  • Spawns worker with correct model + provider
  • Verifies: build → test → lint → guard
  • Commits with correct authorship
  • Tick 7: Self-manages own schedule
  • Runs on per-project cron (15m–360m)

TIER 3 Workers

"The Builder" — 1 per task

  • Spawned via hermes chat -q
  • Reads specified files, writes code
  • Runs tests, fixes compilation errors
  • Returns code changes + verification results
  • Ephemeral — lives only for one task
  • Model selected per-task by foreman
Bane → Supervisor → Foremen → Workers
