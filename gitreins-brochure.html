GitReins

A git-native co-harness that gives your AI agent guardrails. Define tasks, let the agent code, and GitReins judges the result — automatically.

Quick Start →

Workflow

How It Works

Create Tasks

Define criteria via CLI or MCP tools. Each task lists exactly what "done" means: file exists, function present, tests pass, no secrets.

Agent Codes

Your AI agent — Claude, Hermes, Codex, Pi — writes the code. GitReins stays out of the way until it's time to verify.

Agent Completes

Agent calls task.complete. GitReins takes over — the agent's job is done, the harness's begins.

Two-Tier Evaluation

Tier 1: Static guards — secrets, lint, tests — run instantly. Tier 2: An LLM-powered evaluator reads files, checks criteria, returns per-item PASS/FAIL with line numbers.

Commit or Block

Code that passes all gates gets through. Code that doesn't gets blocked — with specific reasons the agent can use to fix it.

Boundaries

What GitReins Doesn't Do

Honest boundaries matter. GitReins is a focused tool — it does one thing well and stays out of the way for everything else.

✗

Doesn't Write Code

GitReins judges code. Your AI agent writes it. There is no code generation, no autocomplete, no refactoring engine.

✗

Not a CI/CD Pipeline

It runs pre-commit, not post-push. It won't deploy your app, run integration tests against staging, or manage releases.

✗

Not a Linter

It coordinates your existing linters — it doesn't replace them. Bring your own ESLint, ruff, or pylint config.

✗

Not a Test Runner

It runs your existing test suite. It won't generate tests, measure coverage, or decide what to test. That's your job.

✗

No Filesystem Sandbox

The evaluator's sandbox is an in-memory dict today. Persistent sandbox with auto-commit to git is designed but not yet built.

✗

No Persistent History

Evaluation results are ephemeral — returned to the caller, not stored. A .gitreins/history/ design exists but isn't implemented.

✗

No Web UI

CLI and MCP only. No dashboard, no admin panel, no browser interface. It's a tool for terminals and agents.

✗

Not Multi-Repo Aware

GitReins lives in one repo. It doesn't coordinate across repos, monorepos, or submodules. Single-repo focus.

✗

Doesn't Replace Your Agent

It's a harness FOR your AI agent — not a replacement. Your agent still thinks, codes, and decides. GitReins verifies.

Get Started

Quick Start

# Clone and install — hooks active in under 10 seconds cd gitreins-poc ./gitreins/install # Create a task with specific criteria python3 gitreins/cli.py task create login-endpoint \ "Implement POST /login" \ "Accepts JSON body" \ "Returns JWT" \ "Has tests" \ "No hardcoded secrets" # Your AI agent writes the code, then completes the task # GitReins evaluates automatically: # Tier 1: secrets scan → lint → tests # Tier 2: LLM reads files, checks each criterion → COMPLETE / INCOMPLETE # Start the MCP server for your AI agent python3 gitreins_mcp/server.py

Requirements: Python 3.10+ · pip install mcp pyyaml requests · An LLM API key for Tier 2 evaluation (DeepSeek, OpenAI, or compatible)
