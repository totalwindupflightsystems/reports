What happens when you take Qwen 3.5 base models and train them to build their own agent scaffolds through reinforcement learning? A 35B model beats a 397B one.

DeepReinforce AI · open weights · MIT license · released June 25, 2026

At a Glance

On June 25, 2026, DeepReinforce AI released Ornith 1.0, a family of four models built on top of existing open-weight bases — Qwen 3.5, Qwen 3.5 MoE, and Gemma 4. What makes Ornith different isn't the architecture. It's the training: self-scaffolding reinforcement learning, where the model learns to generate both solutions and its own task-specific agent scaffolds during RL training.

🔑 The Key Story

Ornith 1.0 proves that training methodology matters more than scale. The 35B MoE Ornith — built on a Qwen 3.5 MoE base — scores 75.6% on SWE-bench Verified, beating Qwen 3.6 Coder (56.6%) by 19 points and even outperforming its own 397B base model on Terminal-Bench. This isn't a bigger model. It's a smarter training recipe.

On Terminal-Bench 2.1, the 35B Ornith (64.2) scores ~20 points higher than the estimated base Qwen 3.5-397B MoE (~44). That's not architecture — that's the RL training gap. The 35B learned to scaffold its own problem-solving, and that skill transfers more effectively than 10× the parameters.

🔬 Two Paths from the Same Base

The Qwen 3.5 lineage split into two coding-focused directions: Qwen 3.6 Coder (fine-tuning for code generation) and Ornith 1.0 (self-scaffolding RL for agentic problem-solving). The results show that RL-based scaffold learning beats fine-tuning by 19 percentage points on SWE-bench at the same parameter tier — the RL path teaches the model how to structure its own problem-solving approach, not just what code looks like.

Self-Scaffolding RL — The Training Innovation

Traditional RL for LLMs follows a simple pattern: the model generates an answer, a verifier scores it, and the model updates to produce better answers. Self-scaffolding RL changes the game: during training, the model generates both a solution and a task-specific scaffold — a structured problem-solving framework it builds for itself.

1️⃣

Generate Scaffold + Solution

The model produces a two-part output: a scaffold (plan, decomposition, tool-use strategy) and a solution (the actual answer or code change). Both are generated in a single forward pass during RL rollouts.

2️⃣

Verify End-to-End

A reward model or execution-based verifier scores the final outcome — does the code pass tests? Did the terminal command succeed? The scaffold is evaluated implicitly through whether the solution works.

3️⃣

RL Optimizes the Whole Pipeline

Gradient updates optimize the model to produce better scaffolds that lead to better solutions. Over training, the model learns which scaffolding strategies work for which problem types — emergent meta-cognition.

🧠 Why This Works

Standard fine-tuning teaches a model what correct code looks like. Self-scaffolding RL teaches a model how to approach problems. The scaffold is the model's internal problem-solving strategy made explicit — and by optimizing it through RL, the model learns strategies that generalize across problem types. This is why a 35B Ornith can beat a 397B base model: the 35B has learned how to think about the problem, not just how to pattern-match solutions.

The approach is particularly effective for agentic coding tasks — SWE-bench, Terminal-Bench — where the path from problem to solution involves multiple steps, tool calls, and strategic decisions. The scaffold acts as an internal "agent harness" that the model constructs on the fly.

What This Means

For Open Agentic Coding

Ornith 1.0 is MIT-licensed with open weights. Combined with the self-scaffolding approach, this means:

🏠

Agentic Coding on Consumer Hardware

The 35B Ornith runs on a consumer GPU and delivers SWE-bench scores that beat proprietary coding models from 2024. The 9B runs on a laptop. This brings frontier agentic coding to individual developers.

🔓

Open Weights, Open Innovation

MIT license means anyone can build on Ornith. The self-scaffolding RL approach is a training recipe that can be applied to future base models. We may see a wave of RL-scaffolded models across architectures.

📐

Scale Is Not the Only Axis

The Ornith results prove that training methodology can deliver gains equivalent to 10× parameter scaling. The research frontier is shifting from "bigger models" to "smarter training."

🔄

A New Training Paradigm

Self-scaffolding RL may become a standard post-training step for agentic models — analogous to how instruction tuning became standard after GPT-3. The question is no longer "how big?" but "how well-trained?"

The Numbers in Context

📊 RL Training Adds ~20 Points on SWE-bench

Across the Ornith family, the self-scaffolding RL adds approximately 15–20 percentage points to SWE-bench Verified scores compared to the base models. Starting from ~55% (Qwen 3.5 base), the RL lifts performance to 69–82% depending on model size. This is a consistent, reproducible gain — not cherry-picking.

The gain is even more dramatic on Terminal-Bench, where the 35B Ornith (~64) roughly doubles the estimated base performance (~30–35 for a 35B base, ~44 for the 397B). Self-scaffolding is particularly powerful for multi-step terminal tasks where strategic planning matters.

⚠️ Caveats & Open Questions

Benchmark versions differ: Qwen 3.6 Coder Terminal-Bench scores are on v2.0, while Ornith uses v2.1. Direct comparison should account for benchmark evolution.

Base model scores are estimates: Exact SWE-bench scores for Qwen 3.5 base models are not published for all sizes; the ~55% figure is a representative baseline.

RL training cost: Self-scaffolding RL requires significant compute during training (generating full rollouts with scaffolds). The tradeoff is training cost vs. inference efficiency — you pay upfront to get a smaller, faster model at runtime.

Generalization beyond code: Most published benchmarks are code-focused. How well self-scaffolding RL transfers to other domains (math, reasoning, creative work) remains to be seen.
