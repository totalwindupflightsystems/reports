🎯 Project Overview

The Mafia AI Benchmark is an autonomous multi-agent benchmarking platform that pits LLMs against each other in the social deduction game Mafia (Werewolf). Its core innovation — the split-pane consciousness paradigm (THINK/SAYS) — separates an AI's internal reasoning from its public declarations, enabling deep analysis of deception, strategy, and emergent behavior across language models.

Built as a pnpm monorepo with 6 workspace packages (apps/server, apps/web, apps/cli, packages/shared, .opencode), deployed via Docker Compose. The legacy game engine (5,303-line battle-tested game logic) runs as a child process bridged to a modern Express + WebSocket server.

Key Design Decisions

  • ✓ Legacy Bridge Pattern — 5,303-line game-engine.js runs as a child process; stdout captured and parsed as JSON events. No engine rewrite needed.
  • ✓ Event Sourcing — All game actions (votes, night actions, phase changes, reveals) persist as timestamped events in SQLite via the addEvent() repository method.
  • ✓ Split-Pane Consciousness — LLM responses carry both THINK (internal reasoning) and SAYS (public declaration) content, stored as AGENT_SAYS_BROADCASTED events.
  • ✓ Multi-Model Support — Configurable LLM provider via llmConfig at game creation time. NeuralWatt primary, OpenRouter fallback.
  • ✓ Docker-native — Two images (server 1.09GB, web 99.5MB), pnpm v11 workspace, health checks, compose orchestration.

🧱 Blocker Log (All Resolved)

Blocker 1: SQLite Native Module (better-sqlite3)

Symptom: pnpm --ignore-scripts skips better-sqlite3 native compilation → Cannot find module 'better-sqlite3'

Fix: Use pnpm v11's pnpm.onlyBuiltDependencies in root package.json + pnpm approve-builds --all to allow specific packages to compile. Rebuild via Docker multistage.

Resolved

Blocker 2: Game Events FOREIGN KEY Constraint

Symptom: Bridge publishes events to EventBus → addEvent() fails with SQLITE_CONSTRAINT — no row in games table for the game_id FK.

Fix: Adapter now inserts the games row via createGame() repository method before spawning the engine child process.

Resolved

Blocker 3: Event Type Mismatch

Symptom: Legacy engine emits MESSAGE, VOTE, ACTION — API expects AGENT_SAYS_BROADCASTED, VOTE_CAST, NIGHT_ACTION_SUBMITTED.

Fix: Type mapping in adapter: 6 bridge event types → 6 canonical event types. Also mapped STATE_CHANGE → GAME_STARTED/GAME_ENDED and REVEAL → MORNING_REVEAL.

Resolved

Blocker 4: Docker ❌ — Shared Package Export Not Found

Symptom: Error [ERR_PACKAGE_PATH_NOT_EXPORTED]: Package subpath './fsm' is not defined by "exports" in @mafia/shared/package.json

Root Cause: tsx resolver + pnpm workspace symlinks skip compiled dist/ — points to source packages/shared/src/ instead.

Fix (3-part): (1) Added raw .ts source paths to shared package exports for tsx compatibility. (2) Restructured Dockerfile build order: install → approve-builds → build shared → build dependents. (3) Used pnpm exec tsx in CMD to ensure tsx is on PATH.

Resolved
