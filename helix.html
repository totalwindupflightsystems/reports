Helix

An open, self-hosted, agent-first software development platform — our alternative to closed, hosted agent platforms (Cursor Origin, Devin Cloud). Built from 8 projects we own plus 8 open-source tools. Built WITH Axiom (the disciplined build pipeline), not built ON Axiom.

The thesis is encoded in the name: two strands — human intelligence and AI intelligence — spiral together through the software development lifecycle, each reinforcing the other. Humans set intent and approve; agents plan, build, review, and merge. Neither operates alone.

Six-Layer Architecture

Helix organizes 16 components into six functional layers. Data flows top-to-bottom (human intent → merged code) and bottom-to-top (observability → human feedback). Each layer is independently deployable but contracts with adjacent layers.

Layer 1
Human Interface
Continue.dev · Cursor · CLI · Telegram (via H4F) — Humans express intent; agents report results
Layer 2
Orchestration
Hivemind (memory+scheduling+IAM) · Kobayashi-Maru (adversarial stress testing) · Helix Dispatcher (spec → work items → agent assignment) — Decompose intent → schedule work → dispatch to agents
Layer 3
Execution
OpenCode (worktree isolation) · Ralph Loop (lock→commit→merge) · Muster (MCP tools for any API) — Agents write code in isolated worktrees, commit with attestation
Layer 4
Git Forge
Forgejo (repos, PRs, CI, Pages) · GitReins hooks (quality gates) · PromptFoo (eval as CI) — Canonical home for all code; branch protection and CI enforced
Layer 5
Quality & Review
Chimera (multi-model formation engine) · Conscientiousness (adversarial eval) · GitReins evaluator (Tier 1+2) · PromptFoo — Nothing merges without multi-model review + adversarial self-eval
Layer 6
Observability & Memory
LangFuse (traces/costs) · DuckBrain (agent memory) · Hivemind (inbox/compiled pattern) — Every decision traceable; every agent remembers across sessions

What We're Building Now (5 Features)

These 5 features complete the platform. All have implementation specifications written — code is being generated by the Helix foreman cron job every 30 minutes.

1. Agent Identity in Forgejo

H4F's known-friends.json → Forgejo OAuth. Agents get real accounts (not bot tokens), SSH keys, scoped permissions (push to feat/* branches, open PRs, cannot merge solo), budget tracking, and trust earned over time. Priority #1 — everything else depends on it.

2. Pre-Flight Cost Estimator

Before Ralph Loop spawns, estimate token burn and ask for budget approval. Tracks per-agent spend, per-PR cost, and enforces budget caps. Integrates with GitReins cache token tracking (v0.4.0+) to distinguish fresh input from 10x cheaper cache hits.

3. Agent-to-Agent PR Negotiation

Two agents disagree in PR comments → Chimera breaks the tie. Multi-model debate with evidence bundles. Agents can veto a merge with evidence. Resolution is traceable via LangFuse.

4. Prompt Registry

prompts/ directory in every repo. GitReins hook verifies every commit links to a prompt version. PromptFoo runs regression tests on every prompt change. Version-controlled, attested prompts — no more "what prompt generated this code?"

5. Agent Marketplace

Discoverable registry of pre-built agents with reputation scores. Agents earn trust over time based on successful merges, review quality, and human feedback. Ranked, searchable, self-improving.

Specs written: All 5 features have exhaustive implementation specifications (2,745 total lines across agent-identity, cost-estimator, pr-negotiation, prompt-registry, and agent-marketplace specs). Go stubs exist for Features 2–5. Feature 1 has 1,769 lines of Go code with typed errors, state files, dry-run infra, and rate limiters.

The Four Glue Primitives

Everything in Helix slots into these four primitives. Understanding them is prerequisite to understanding any integration.

Execution
Ralph Loop
Acquire lock → create worktree → write code → commit with attestation → open PR → merge → release lock. Guarantees no two agents edit the same branch concurrently; every commit carries provenance.
API Compatibility
Muster
Parse OpenAPI spec → generate MCP tools + CLI commands + shell completions. Any REST API becomes agent-callable in minutes — no hand-written wrappers.
Quality Gates
GitReins
Tier 1 (static: secrets, lint, tests, build) + Tier 2 (agentic LLM evaluator with 9 tools). Blocks commits that fail objective checks before they enter the forge.
Agent Hosting + Identity
Hermes4Friends
Per-agent Docker sandbox with budget, permissions, VPN, and known-friends identity. Agents run in isolation; identity is provisioned and scoped.

Architectural Principles

Agents have real identities
Every agent is a first-class Forgejo user with SSH keys, scoped permissions, and a budget. Agents are not anonymous API calls — they are accountable actors.
Multi-model agreement required
A single model's review is insufficient. Chimera formations require at least two independent models plus an aggregator. Conscientiousness adds adversarial self-evaluation.
Everything is traceable
Every agent action produces a LangFuse trace with model, tokens, cost, latency, and decision rationale. Every commit carries an attestation linking back to the prompt that generated it.
Sandbox by default
Agents execute in Docker containers with network isolation (gluetun VPN). No agent touches the host filesystem directly. Budgets enforced per-agent.
Self-hosted, no lock-in
Every component can run on a single Hetzner box. The only external dependencies are LLM APIs, abstracted behind Chimera's provider gateway.
Loops, not prompts
Agents are driven by well-designed feedback loops — not hand-tuned prompts. Ralph Loop, GitReins gates, and Chimera review constrain agent behavior structurally.
