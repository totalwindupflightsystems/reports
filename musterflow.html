One Product.
One Surface.

A self-hosted developer platform that turns any API into a CLI, an MCP tool, and a workflow — with a community-contributed catalog that grows with every user. Built on Muster. Inspired by Context7.

Inspiration

The Context7 Model

Context7 (by Upstash) is an MCP server that gives AI agents up-to-date library documentation. Its catalog is community-contributed: anyone submits docs for a library, it gets scored, and every AI agent that installs Context7 gets access to it. Missing a library? You submit it. Now everyone has it.

This is the model for our product — but for APIs instead of libraries.

# Deploy
$ brew install musterflow
$ musterflow start
→ Dashboard running on http://localhost:9876
→ MCP endpoint: http://localhost:9876/mcp
→ CLI ready. Try: musterflow connect <url>

# Connect an API
$ musterflow connect https://api.github.com
→ Validated OpenAPI spec
→ Generated 42 CLI commands (gh issues list, gh repos create, …)
→ Registered 42 MCP tools
→ Added to your catalog. Push to community? [Y/n]

# Use it
$ musterflow gh issues list --repo cli/cli --state open --limit 5
→ 5 issues returned. Formatted as table.

# Chain APIs into a workflow
$ musterflow flow create
→ on stripe:charge.succeeded → gh issues create(title="Payment: $amount")
→ Workflow deployed. Triggered by webhook.

# Your AI agent now has all these tools
$ musterflow mcp endpoint
→ http://localhost:9876/mcp
# Paste that into Claude Desktop, GPT, or Hermes.
Surfaces

One Product, Four Ways to Use It

🖥 Web Dashboard

Connect APIs, browse the community catalog, manage workflows, view audit logs. The hub for everything.

⌨ CLI

Every connected API becomes a native CLI with typed subcommands, bash/zsh/fish completion, and pipe-able output.

🤖 MCP Endpoint

Every connected API is an MCP tool. AI agents (Claude, GPT, Hermes) can call any API without integration code.

⚡ Workflow Engine

Chain APIs with Starlark DSL. Webhook triggers. WASM transforms. "When X happens in API A, do Y in API B."

How Contributions Work

  1. Anyone can add an API. Paste an OpenAPI URL. It validates. It's immediately available to you. Push to the community catalog with one click.
  2. Anyone can share a workflow. Write a Starlark script that chains APIs. Push it. Others install it. Works out of the box if they have the same APIs connected.
  3. Anyone can publish a WASM transform. Request/response middleware. PII redaction. Data reshaping. Push it. Others install it.
  4. Quality scoring surfaces the good ones. Official specs score high. Community-inferred specs get marked. Untested connectors are flagged. The community curates by using.

No PR approval. No gatekeeping. The catalog is a public repo (or registry). Quality emerges from usage, not from review.

Positioning

Why This Wins

No SDK tax

Every API integration today requires: find the SDK, install it, read the docs, write the code. Here: paste a URL. Done.

AI-native from day one

Every connected API is automatically an MCP tool. AI agents can call any SaaS. No integration code. No middleware.

Self-hosted = trust

Your API keys. Your data. Your machine. Nothing phones home. No vendor lock-in. Open source.

Community = network effects

Every new API connector makes the platform better for everyone. Every shared workflow is free leverage.

One install

Not "install the CLI, then the MCP server, then the workflow engine." One binary. One brew install. One docker run.

Progressive disclosure

Start with the CLI (any dev understands it). Discover the MCP endpoint. Discover workflows. Discover the catalog. No learning cliff.

MVP

What Ships First

The minimum viable product that proves the model:

  1. Single binary — embeds Muster, a lightweight web dashboard, and the MCP endpoint. brew install and docker run.
  2. API connection — paste an OpenAPI URL. It validates. It generates CLI commands and MCP tools immediately.
  3. CLI surface — every connected API becomes typed commands with autocomplete. musterflow gh issues list. musterflow stripe charges list.
  4. MCP endpoint — local endpoint that AI agents connect to. Every connected API is a tool.
  5. Catalog push/pull — push your connected APIs to the community catalog (a public GitHub repo). Pull from it. Search it from the dashboard.
  6. Quality scores — official specs, community-inferred, untested. Badges. Sorting.

Phase 2 (month 2): Workflow engine (Starlark DSL + webhook triggers). WASM transform publishing.
Phase 3 (month 3): Multi-tenancy. Team shared vaults. Audit logging. Enterprise features.

Design Constraint

The 30-Second Rule

The demo must make someone say "I need this" in under 30 seconds, without knowing what Muster is. Muster is the engine — not the headline. The headline is:

"Paste any API URL. Get a CLI, MCP tools, and workflows. Instantly."
