1. Vision

Every SaaS company ships an API. Every developer spends hours wiring those APIs into CLIs, chatbots, and automation scripts. MusterFlow eliminates that work: paste an OpenAPI URL, get a typed CLI, MCP tools for AI agents, and a workflow engine — instantly.

The 30-second demo:
brew install musterflow → musterflow start → paste https://api.github.com → musterflow gh issues list --repo cli/cli works. Paste Stripe. musterflow flow create → on stripe:charge.succeeded → gh issues create(title="Payment: $amount"). They tweet.

Why now: AI agents (Claude, GPT, Hermes) are bottlenecked on API integration. Every new API requires custom integration code. MCP protocol standardizes tool calling — MusterFlow makes every API an MCP tool without code. The Context7 model proved community-contributed catalogs work for docs. MusterFlow applies the same model to API execution.

2. Product

2.1 Four Surfaces, One Binary

🖥 Web Dashboard

Connect APIs, browse community catalog, manage workflows, view audit logs. Self-hosted, local. Dark theme, responsive.

musterflow start → localhost:9876

⌨ CLI

Every connected API becomes typed subcommands with shell completion. Table, JSON, CSV, JSONL, Parquet output.

musterflow gh issues list --state open

🤖 MCP Endpoint

Every connected API is an MCP tool. AI agents (Claude, GPT, Hermes) call any API without integration code.

http://localhost:9876/mcp

⚡ Workflow Engine

Chain APIs with Starlark DSL. Webhook triggers. WASM transforms. Community-published workflows.

musterflow flow create → on stripe:charge → gh issues create

3. Architecture

3.1 Engine: Muster

MusterFlow imports the Muster engine (github.com/wojons/muster) as a Go library. Muster provides the core primitives — MusterFlow adds packaging, dashboard, catalog, and workflow UX.

Muster PackageWhat It DoesMusterFlow Usage
pkg/openapiParse & validate OpenAPI specs, $ref resolutionConnect, refresh, catalog pull
pkg/generatorGenerate cobra CLI commands from specsLazy command generation
pkg/mcpMCP server, protocol, tool descriptorsHTTP MCP endpoint
pkg/requestHTTP request builderAll API calls
pkg/authOAuth2 flows, token managementAuth login command
pkg/wasmwazero-based WASM runtimeTransform execution (Phase 2)
pkg/dslStarlark interpreterWorkflow DSL (Phase 2)

3.3 Data Flow

User pastes OpenAPI URL
  → openapi.Parse() validates spec
  → generator.GenerateCommands() builds cobra tree
  → Registry stores connection in DuckDB
  → CLI: musterflow <api> <op> --flag=val
  → request.Builder.Execute() makes HTTP call
  → response formatted (table/JSON/CSV/JSONL/Parquet)

MCP path:
  → handlers.Registry registers tools from connected APIs
  → HTTP JSON-RPC server bridges to /mcp endpoint
  → AI agent calls tools/call → ExecuteCommand → HTTP → formatted response

Workflow path (Phase 2):
  → Starlark DSL compiles to workflow steps
  → Webhook trigger dispatches execution
  → WASM transforms process data between steps

5. Competitive Landscape

ProductWhat It DoesMusterFlow vs
Context7 Community-submitted library docs for AI agents Context7 is read-only (docs retrieval). MusterFlow is execute (CLI + MCP tools + workflows). Same community model, different substrate.
OpenAPI Generator Generate SDK clients from OpenAPI specs Requires code generation, build step, language-specific output. MusterFlow is instant — no codegen, no build, any language via CLI + MCP.
Postman GUI API client, collections, environments SaaS, GUI-first. MusterFlow is CLI-first, self-hosted, AI-agent-native via MCP. Postman is for humans; MusterFlow is for humans and agents.
n8n / Zapier Visual workflow automation SaaS, GUI drag-and-drop, proprietary. MusterFlow workflows are code (Starlark), version-controlled, self-hosted. WASM transforms are community-published.
Pipedream API integration platform SaaS, JavaScript-based. MusterFlow is self-hosted, Go binary, Starlark DSL. No vendor lock-in.
MCP.so / Smithery MCP server directories Requires per-API MCP server code. MusterFlow generates MCP tools from any OpenAPI spec with zero code. Catalog is a GitHub repo, not a proprietary directory.

5.1 Moat

  • Zero-code MCP tool generation. No other product turns an OpenAPI URL into working MCP tools without writing server code.
  • Community catalog as GitHub repo. Not a proprietary directory — forkable, PR-driven, no registration. Teams can run their own catalog.
  • One binary, four surfaces. CLI + MCP + dashboard + workflows from a single brew install. No split installs, no SaaS dependency.
  • Self-hosted, MIT license. User's keys, user's machine, user's data. Nothing phones home.

6. Risks & Mitigations

RiskSeverityMitigation
OpenAPI spec quality varies wildly across APIs High Quality scoring system (domain reputation + spec structure). Community flags bad specs. Muster parser handles common spec issues.
Cold start — empty catalog at launch High Seed with 10 high-value APIs (GitHub, Stripe, Slack, Discord, OpenAI, Notion, Linear, Jira, Twilio, Cloudflare). The value is immediate even with one API.
API auth complexity (OAuth2, mTLS, custom schemes) Medium Muster engine already handles all 4 auth types. OAuth2 flow skeleton complete. Keychain integration for OS-native secret storage.
MCP protocol still evolving Medium HTTP JSON-RPC works today. Stdio transport added in Phase 2. Protocol updates are additive.
Competition from well-funded tools (Postman, Zapier) Low Different market: self-hosted, AI-agent-native, open-source. Postman/Zapier are SaaS for humans. MusterFlow is infrastructure for agents.
WASM transform security (untrusted community code) Medium wazero sandbox with deny-network default. Capability negotiation. Community audit surface (GitHub PR review).
Muster engine dependency (single maintainer) Medium Muster is MIT-licensed, 56 packages, full test coverage. MusterFlow imports it as a library — can be vendored. Generator, parser, MCP server are the critical paths.
