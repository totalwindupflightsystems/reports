⟳ The Autonomous Loop

Every 6 hours, the steward cron wakes. It reads the task queue, picks the top pending item, researches, codes, runs guards, commits, pushes, and marks it done. It never asks questions. It never waits for approval. It acts.

⏰ Cron wakes → 📋 Read tasks.md → 🔬 Research → 💻 Code → 🛡️ Guard → 📦 Commit+Push → ✅ Mark done

🛡️ Quality Harness — Three Tiers

Every change passes through three independent quality gates. No tier is optional.

4. Property-Based Testing

5. Cluster Fault Injection

Jepsen-style: network partitions, primary kills during backfill, split-brain, disk-full behavior.

⚖️ How GitReins Enforces Quality

Every task becomes a GitReins task with criteria the LLM evaluator mechanically verifies. No human judgment — file:line evidence or it didn't happen. 

# Example: "Add vector index support"
- id: vec-idx-1
  title: "HNSW index data structure"
  criteria:
    - "src/rdb_protocol/geo/indexing.hpp defines HnswIndex class"
    - "src/unittest/geo_indexes.cc tests HNSW insert+search k-NN"
    - "cppcheck src/rdb_protocol/geo/ --enable=all returns 0 new warnings"
    - "clang-tidy checks='bugprone-*' returns 0 new warnings"
