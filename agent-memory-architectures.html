Holographic Memory vs DuckBrain

A comparison of two agent memory architectures — one inspired by neuroscience, the other by DevOps. June 2026.

The Contenders

🧠 Hermes Holographic Memory

Built into Hermes Agent by Nous Research. Uses HRR (Holographic Reduced Representations) algebra — memories are superposed complex-valued vectors with algebraic recall. Local SQLite, fully air-gapped, trust-scored.

📦 DuckBrain

Git-native memory bank built by totalwindupflightsystems. JSONL files in version-controlled git repos, pgvector for vector search, FTS5 for keyword search, hierarchical keys, namespace isolation, soft-delete with compaction.

Where They Converge

Both solve the same fundamental problem: how does an AI agent remember things across sessions without an external API?

Both are local, air-gapped, and agent-native. Both store structured facts with metadata. Both support querying by semantics and keywords. Both are designed for long-running autonomous agent loops where context windows don't survive across sessions.

Where They Diverge

Philosophy

Hermes Holographic takes the neuroscience-inspired route. Memories are distributed patterns across a vector space. No single neuron stores a memory — the memory is the pattern. HRR algebra means recall is a mathematical operation on superposed vectors, not a database lookup. You get approximate recall, and it degrades gracefully under load — more memories means fuzzier results, not a hard failure.

DuckBrain takes the DevOps route. Every memory is a line in a JSONL file, version-controlled in git. You get exact retrieval with full provenance — who wrote this, when, from which session, verified against which source. The trade-off is that it doesn't degrade gracefully — git repos have size limits, JSONL scanning has linear cost, and compaction is a batch operation.

When the math matters

Hermes Holographic gives you "what's the closest thing to what I'm looking for?"
DuckBrain gives you "give me every fact about ASCE from June 2026, sorted by recency."

When the audit trail matters

DuckBrain's git-native architecture means every single memory has a commit hash, author, timestamp, and diff. If an agent hallucinates a "fact" or writes stale data, you can git log to find exactly when it was written and by whom. Hermes Holographic has no equivalent — memories are superposed in vector space with no per-entry provenance.

Our Stack: How We Use DuckBrain

We run DuckBrain as an MCP server with 12 tools, serving 400+ entries across two namespaces (hermes-memory and frontiers-ghost). Three crons keep it alive:

  • Context-Sync (every 6h) — scans Hermes sessions, GitHub, GitLab, Obsidian vault. Extracts durable facts with traceability metadata.
  • Deep Reflection (every 12h) — synthesizes patterns across sessions, flags stale entries, auto-patches skills on recurring errors.
  • Dreaming (nightly at 3am) — synthetic memory generation, associative linking between project entries.

All entries carry 6+ attributes: certainty, severity, source_type, source_ref, verification method, and discovery method. Every fact is traceable back to a specific session, git commit, or file.

Which One Should You Use?

Use Hermes Holographic when…

  • You want zero setup — it's built into Hermes Agent
  • You have high volume of unstructured memories (conversation logs, thoughts)
  • You don't need audit trails or per-entry provenance
  • You want graceful degradation under scale

Use DuckBrain when…

  • You need exact retrieval with full provenance
  • You run multiple agents that need isolated memory spaces
  • You want git-native version control — every memory is reviewable
  • You need structured, queryable metadata per entry
  • You're running autonomous agent loops that need cron-driven memory maintenance

Bottom Line

They're not competitors — they solve different points on the spectrum of memory fidelity vs recall flexibility. Hermes Holographic gives you approximate recall with elegant math and zero overhead. DuckBrain gives you exact retrieval with full audit trail and DevOps-style maintenance.

We chose DuckBrain because we run 20+ autonomous cron loops across 14 projects. When an agent claims "ASCE build passes" at 3am, we need to trace that fact back to the exact session that verified it — not get a fuzzy vector match.
